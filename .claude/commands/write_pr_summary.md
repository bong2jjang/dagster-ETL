# PR 요약 작성

`git log` 명령으로 현재 브랜치의 커밋 이력을 확인합니다. 베이스 브랜치(main)와의 `git diff`로 전체 변경 사항을 확인합니다. 이 컨텍스트와 세션에서 알고 있는 변경 내용을 바탕으로 아래 형식의 PR 요약을 마크다운으로 작성합니다.

```md
## 요약 및 동기

[변경 내용의 간략한 요약. 새 API가 추가된 경우 코드 샘플 포함.]

## 테스트 방법

[간략한 테스트 계획. 예: "신규 단위 테스트 추가.", "기존 테스트 스위트 통과 확인.", "로컬 dagster dev로 검증." 한 문장으로 작성 가능.]

## 변경 로그

[사용자에게 영향을 미치는 변경 사항이 있을 경우 기재. 내부 리팩토링만인 경우 이 섹션 제거.]
```

불릿 포인트는 절제하여 사용하고, 이탤릭/볼드 과다 사용을 피합니다. 짧은 문장을 사용합니다. 사용자가 복사하여 PR에 붙여넣을 수 있도록 마크다운을 출력합니다.
